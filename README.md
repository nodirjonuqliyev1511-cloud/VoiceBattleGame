# Voice Battle

Игра для ведущего: участники орут в микрофон телефонов, ведущий видит гонку техники
(машина/джип/трактор/танк) в реальном времени — кто громче орёт, тот вырывается вперёд.

Теперь с системой логина: **ты — администратор**, создаёшь аккаунты ведущим в админке,
они заходят каждый под своим логином/паролем. Участники по-прежнему заходят просто по QR,
без регистрации.

## Запуск локально (для вечеринки в своём Wi-Fi)

```
npm install
npm start
```

В терминале появятся адреса. Главные:
- `https://localhost:3000/admin-login.html` — вход в админку (это ты)
- `https://localhost:3000/host-login.html` — вход для ведущих
- Участники заходят через QR-код, который показывается ведущему после входа

### Первый вход в админку

По умолчанию логин/пароль администратора — `admin` / `admin123`. Обязательно смени их
(см. раздел "Переменные окружения" ниже) перед тем, как давать кому-то ссылку на игру.

### Как завести ведущего

1. Зайди в админку (`/admin-login.html`)
2. В форме "Новый ведущий" укажи логин (латиницей), имя для афиши (то, что увидят
   участники как "Host: Имя") и пароль
3. Нажми "Создать ведущего" — готово, эти логин/пароль отдаёшь ведущему
4. Ведущий заходит на `/host-login.html` под этими данными — дальше всё как раньше
   (выбор города, количество игроков, QR для участников)

## Переменные окружения

Задаются либо в файле `.env` (локально — создай сам, не входит в комплект) через
`process.env`, либо в панели Render (см. ниже). Все опциональны, но их стоит задать:

| Переменная | Зачем | Если не задана |
|---|---|---|
| `ADMIN_USERNAME` | логин администратора | `admin` |
| `ADMIN_PASSWORD` | пароль администратора | `admin123` |
| `SESSION_SECRET` | ключ для подписи cookie-сессий | случайный при каждом запуске (все разлогинятся при перезапуске сервера) |
| `PUBLIC_BASE_URL` | публичный адрес игры для QR-кода (для Render) | используется локальный IP в Wi-Fi |

Локально переменные можно выставить прямо в терминале перед запуском, например
(Windows, PowerShell):
```
$env:ADMIN_USERNAME="дмитрий"; $env:ADMIN_PASSWORD="секретный-пароль"; npm start
```

## Деплой на Render

1. Залей проект в GitHub-репозиторий (обычный `git init`, `git add`, `git commit`,
   `git push` — папку `node_modules` заливать не нужно, она в `.gitignore`).
2. На [render.com](https://render.com) → New → Web Service → подключи репозиторий.
3. Настройки сервиса:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
4. В разделе **Environment** добавь переменные:
   - `ADMIN_USERNAME`, `ADMIN_PASSWORD` — свои логин/пароль администратора
   - `SESSION_SECRET` — любая длинная случайная строка
   - `PUBLIC_BASE_URL` — адрес, который Render выдаст сервису
     (вида `https://voicebattle.onrender.com`, без слэша на конце; узнаётся после
     первого деплоя, можно проставить и передеплоить)
5. Render сам выставляет переменную `RENDER=true` — по ней сервер понимает, что он
   в облаке, и не пытается делать свой самоподписанный HTTPS-сертификат (Render уже
   даёт настоящий HTTPS сам). Ничего дополнительно включать не нужно.
6. После деплоя участники заходят по обычному QR/ссылке через интернет — телефоны
   ведущего и гостей уже не обязаны быть в одном Wi-Fi.

**Важно про аккаунты ведущих на Render:** обычный (не платный) веб-сервис Render
использует временную файловую систему — она сбрасывается при каждом новом деплое
(не при обычном "засыпании"/перезапуске, а именно при новой публикации кода). Это
значит, что аккаунты ведущих, созданные через админку, **пропадут при следующем
деплое** — придётся создать их заново. Для вечеринки это обычно не проблема (создал
перед событием). Если нужно, чтобы аккаунты переживали каждый деплой — на Render
подключается платный Persistent Disk с точкой монтирования на папку `data/`, либо
аккаунты стоит перенести во внешнюю базу данных.

## Предупреждение браузера "соединение не защищено" (только локальный режим)

При запуске локально (не на Render) сервер использует HTTPS с самоподписанным
сертификатом — это обязательное требование браузеров для доступа к микрофону.
Из-за самоподписанного сертификата при первом заходе браузер один раз показывает
предупреждение — это нормально, нужно нажать "Дополнительно" → "Перейти на сайт".
**На Render этого предупреждения не будет вообще** — там настоящий HTTPS-сертификат
от самой платформы.

### Как убрать это предупреждение совсем

Если хочется, чтобы гости заходили без единого лишнего клика — можно пробросить сервер
через бесплатный туннель Cloudflare, у которого уже настоящий доверенный сертификат:

1. Скачай `cloudflared` с официального сайта Cloudflare (раздел Cloudflare Tunnel,
   есть версия под Windows).
2. Запусти сервер как обычно (`npm start`), оставь окно открытым.
3. В новом окне терминала выполни:
   ```
   cloudflared tunnel --url https://localhost:3000 --no-tls-verify
   ```
4. Cloudflared выдаст ссылку вида `https://случайные-слова.trycloudflare.com`.
5. Открой экран ведущего по адресу вида
   `https://localhost:3000/host.html?tunnel=https://случайные-слова.trycloudflare.com`
   (вставь свою ссылку из шага 4 после `?tunnel=`) — QR-код автоматически перестроится
   на эту ссылку, и предупреждение у гостей исчезнет.

Важно: для этого способа нужен интернет на компьютере ведущего (сам туннель идёт через
интернет, даже если участники физически рядом). Если интернета нет или не хочется
возиться с туннелем — просто используй обычный QR, предупреждение нажимается один раз
за вечер на каждом телефоне.

## Видео-фоны городов

Верхнюю полосу с городом во время гонки можно заменить настоящим видео вместо
анимации из блоков — по одному видео на каждый город.

1. Сделай (или найди) короткий зацикленный ролик для каждого города — важно,
   чтобы первый и последний кадр совпадали, иначе при зацикливании будет
   заметный скачок/пауза. Формат: mp4 (H.264), без звука можно, разрешение
   любое (растянется на всю ширину баннера — оптимально что-то широкое,
   16:9 или шире), длительность 5-15 секунд обычно достаточно.
2. Назови файлы ровно так и положи в папку `public/videos/`:
   - `tokyo.mp4` — Неоновый Токио
   - `newyork.mp4` — Ночной Нью-Йорк
   - `tashkent.mp4` — Ночной Ташкент
   - `vegas.mp4` — Лас-Вегас ночью
3. Всё, больше ничего настраивать не нужно — при старте гонки игра сама
   подставит видео нужного города. Если файла для какого-то города нет —
   останется прежняя анимация из блоков, ничего не сломается.

## Настройки

В `server.js`:
- `RMS_DB_FLOOR` / `RMS_DB_CEIL` — чувствительность шкалы громкости к микрофону
- `COUNTDOWN_SECONDS` — отсчёт "приготовьтесь" перед стартом (сейчас 3)
- `DISTANCE_KM_PER_SECOND_AT_MAX` / `FINISH_DISTANCE_KM` — насколько быстро
  растёт "пройденная дистанция" и сколько нужно проехать до финиша
- `CAR_COLORS` / `VEHICLE_TYPES` — палитра цветов и набор типов техники,
  которые по кругу раздаются подключившимся игрокам
